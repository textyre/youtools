# Subtitles Extraction & MCP Integration — Design Document

**Date**: 2026-03-02
**Status**: Approved

## Summary

Add subtitle extraction from YouTube videos via yt-dlp, a JSON config system with defaults, and an MCP server exposing 5 tools for AI agent integration (subtitle retrieval, language listing, search, video info, and summarization).

## Decisions

| Decision | Choice |
|----------|--------|
| Subtitle extraction | yt-dlp (external binary) |
| Subtitle formats | All yt-dlp formats: srt, vtt, json3, srv1, srv2, srv3, lrc, txt |
| Config format | JSON at `~/.config/youtools/youtools.json` |
| Default save path | `~/youtools/subtitles/` |
| MCP framework | `@modelcontextprotocol/sdk` (official TypeScript SDK) |
| MCP tools | 5: get_subtitles, list_subtitle_languages, search_in_subtitles, get_video_info, summarize_video |
| Summary generation | Delegate to LLM agent (return subtitles + prompt instruction) |
| Architecture | Approach A: subtitles module + separate MCP entry point |

## Configuration

File: `~/.config/youtools/youtools.json`

```json
{
  "subtitles": {
    "outputDir": "~/youtools/subtitles",
    "defaultFormat": "srt",
    "defaultLanguage": "en",
    "preferAutoGenerated": false
  },
  "cache": {
    "ttl": 3600,
    "dir": "~/.cache/youtools"
  }
}
```

Behavior:
- Lazy-loaded on first access
- Deep-merged with hardcoded defaults
- CLI flags override config values
- Exported as typed `AppConfig` object

## CLI Command

```
youtools subtitles <VIDEO_ID> [options]

Options:
  -f, --format <fmt>       Subtitle format (srt|vtt|json3|srv1|srv2|srv3|lrc|txt) (default: from config)
  -l, --lang <code>        Subtitle language code (default: from config)
  --auto                   Prefer auto-generated subtitles
  --list                   List available languages/tracks without downloading
  --search <query>         Search text in subtitles with timestamps
  -o, --output <path>      Output path (default: from config)
  --stdout                 Print subtitles to stdout instead of saving to file
```

Data flow:
1. Parse CLI args, merge with config
2. Call `yt-dlp --list-subs` to get available tracks
3. If `--list` → display table and exit
4. Download via `yt-dlp --write-sub --sub-lang <lang> --sub-format <fmt> --skip-download`
5. If `--stdout` → print to terminal; if `--search` → filter by query with timestamps
6. Otherwise save to `outputDir/<videoId>.<lang>.<fmt>`

File naming: `<videoId>.<lang>.<format>` — e.g. `dQw4w9WgXcQ.en.srt`

## MCP Server

Entry point: `src/mcp.ts` — runs as `node dist/mcp.js` via stdio transport.

### Tool 1: `list_subtitle_languages`
- **Input**: `{ videoId: string }`
- **Output**: JSON array of available tracks `[{ lang, name, isAutoGenerated }, ...]`
- Uses `yt-dlp --list-subs --print-json`

### Tool 2: `get_subtitles`
- **Input**: `{ videoId: string, lang?: string, format?: string }`
- **Output**: Subtitle text in chosen format. For `txt` — clean text without timestamps.
- Caches in `outputDir` (reads from disk if file exists)
- Default `lang` and `format` from config

### Tool 3: `search_in_subtitles`
- **Input**: `{ videoId: string, query: string, lang?: string }`
- **Output**: JSON array of matches `[{ timestamp, text }, ...]`
- Downloads subtitles, parses timestamps, filters by query (case-insensitive)

### Tool 4: `get_video_info`
- **Input**: `{ videoId: string }`
- **Output**: JSON with metadata (title, channel, duration, description, upload_date, view_count)
- Uses `yt-dlp --dump-json --skip-download`

### Tool 5: `summarize_video`
- **Input**: `{ videoId: string, lang?: string, format?: string }`
  - `format` — summary style: `"brief"` (default), `"detailed"`, `"checklist"`, `"custom:<instructions>"`
- **Output**: Subtitle text + prompt instruction for the agent
  - Default prompt (`brief`): "Make a brief summary, highlight 2-3 main ideas, create 5-10 checklist items"
  - The LLM agent generates the summary from subtitles and the prompt

### MCP Client Configuration Example (Claude Code)
```json
{

```

## Error Handling

- **yt-dlp not installed**: Check at startup (`which yt-dlp`), clear error message with install instructions
- **Subtitles unavailable**: Return error listing available languages
- **Invalid videoId**: Validate format before calling yt-dlp
- **yt-dlp timeout**: 30 seconds default, configurable via config
- **File system errors**: Graceful fallback (can't write → output to stdout)
